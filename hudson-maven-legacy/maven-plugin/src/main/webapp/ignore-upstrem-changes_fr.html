Si cette case est cochée, Hudson parcourera les fichiers POM de ce projet et verra si une des dépendances SNAPSHOT sont construites également sur ce serveur Hudson. Dans ce cas, Hudson positionnera une relation dépendance de build, de façon à ce que, à chaque fois que le job dont il dépend est construit et qu'un nouveau jar SNAPSHOT est créé, Hudson programme un nouveau build pour ce projet.

Cela est pratique pour obtenir une intégration continue automatique. Hudson vérifiera que les dépendances SNAPSHOT de l'élément <dependency> dans le POM, ainsi que les <plugin>s et les <extension>s utilisés dans les POMs.

Si ce comportement est problématique, décochez cette option.
